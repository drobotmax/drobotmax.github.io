AI-автоматизация операций WB-селлера

Альберт - разработка системы под ключ
ЗаказчикИП Альберт Л.
ИсполнительИП Дробот М.В.
ФорматРазработка под ключ
Срок2 календарные недели
Стоимость150 000 руб
Система AI-автоматизации операций WB-селлера. Парсер данных Wildberries и аналитики маркетплейсов автоматически заполняет таблицу R&P по магазинам, считает ключевые метрики и присылает алерты в Telegram. Контроль поставок и шаблонные сообщения поставщикам - в одном контуре. Поиск интересных товаров и аналогов на 1688 запускается по команде и сразу заполняет Юнит-Экономику. AI-ассистент по SKU предлагает решения на основе ретроспективы менеджеров и анализа конкурентов - менеджер подтверждает или корректирует. Финал - передача исходного кода, доступов и документации.
Состав поставки

5 модулей. Каждый сдаётся отдельно, финальная приёмка - по всему составу

01
Discovery + ТЗ
Доступы, бизнес-правила, архитектура
02
R&P + дашборд
Автозаполнение, метрики, алерты
03
Контроль поставок
Трекер сроков, шаблоны поставщикам
04
Парсинг товаров
WB + 1688 + Юнит-Экономика
05
AI-рекомендации
Ассистент по SKU, лог решений
Модуль 01 Discovery и фиксация ТЗ
Состав работ
  • Инвентаризация доступов к информационным системам Заказчика: Wildberries Seller API, аналитические сервисы маркетплейсов, рекламные кабинеты, бидер, рабочие таблицы, источники данных по поставщикам
  • Фиксация бизнес-правил отбора товаров и решений менеджеров: управление ценой, рекламной кампанией, остатками, выкупами
  • Архитектурное решение по инфраструктуре системы
  • Согласование критериев готовности по каждому из последующих модулей
Поставка
Документ ТЗ + перечень доступов + архитектурная схема системы
Приёмка
Заказчик утверждает ТЗ перед стартом разработки модулей 2-5
Модуль 02 R&P-автозаполнение и базовый дашборд
Состав работ
  • Парсер Wildberries Seller API → нормализация данных → ежеутреннее автообновление таблицы R&P по магазинам Заказчика
  • Подтяжка рекламных расходов из рекламного контура
  • Дашборд по ключевым метрикам: ДРР, CTR, конверсии, выкупы, оборачиваемость
  • Алерты в Telegram ответственному менеджеру по правилам, согласованным в ТЗ
Поставка
Работающий пайплайн в проде, исходный код, конфигурация алертов, инструкция по эксплуатации
Приёмка
Заказчик подтверждает корректность данных за 3 рабочих дня сравнением с ручной таблицей
Модуль 03 Контроль поставок
Состав работ
  • Трекер сроков поставок по контуру Заказчика
  • Алерты в Telegram ответственному менеджеру при приближении к срокам и просрочках
  • Автогенерация шаблонных сообщений поставщикам
Поставка
Трекер + бот алертов + шаблоны сообщений, исходный код, инструкция
Приёмка
Заказчик проверяет работу на реальных поставках в течение калибровочного периода
Модуль 04 Парсинг товаров и поиск аналогов
Состав работ
  • Выгрузка категорий товаров из аналитического сервиса маркетплейса
  • Отбор товаров по критериям, зафиксированным в ТЗ
  • Получение карточек и фотографий товаров с Wildberries
  • Поиск аналогов по фото на 1688 и сбор данных по топ-продавцам
  • Заполнение таблицы Юнит-Экономики данными по найденным товарам
Поставка
Пайплайн «по команде» + интеграция с таблицей Юнит-Экономики, исходный код, инструкция
Приёмка
Прогон на согласованной категории, сравнение результата с ручной выборкой менеджера
Модуль 05 AI-рекомендации по SKU
Состав работ
  • Логика рекомендаций по SKU на основе ретроспективы решений менеджеров (данные из Модуля 2) и анализа конкурентов (из Модуля 4)
  • Параллельная работа «менеджер + AI» на калибровочном периоде
  • Лог решений и метрика сходимости рекомендаций AI с фактическими решениями менеджеров
  • Постепенное расширение зоны автономии AI по мере подтверждения качества
Поставка
AI-ассистент по SKU, исходный код, инструкция, лог решений с метрикой сходимости
Приёмка
Прогон на согласованной выборке SKU за калибровочный период, сравнение AI-рекомендаций с фактическими решениями менеджеров
Формат работы

Разработка под ключ, не интенсив

Исполнитель ведёт разработку самостоятельно. Заказчик участвует в постановке задач, демо промежуточных результатов и приёмке.

Как идёт работа

  • Самостоятельная разработка Исполнителя в течение всего срока
  • Рабочие созвоны 1-2 раза в неделю для синхронизации, демонстрации промежуточных результатов и приёмки модулей
  • Оперативная коммуникация в Telegram в рабочее время (10:00-18:00 МСК)
  • Параллельная приёмка модулей: Модуль N сдан → Заказчик проверяет → Исполнитель параллельно работает над N+1
  • Календарь работ ведёт Исполнитель и согласовывает контрольные точки с Заказчиком
Что не входит в работы Исполнителя

Границы поставки

  • Платные тарифы и API сторонних сервисов (аналитика маркетплейса, LLM-провайдеры, инфраструктура) - оплачивает Заказчик
  • Развитие и сопровождение системы после подписания акта сдачи-приёмки - оформляется отдельным договором
  • Обучение персонала Заказчика по работе с маркетплейсами и закупками - программа касается только переданного ПО
Финальная приёмка

Что получает Заказчик по итогу

  • Исходный код всех разработанных программных модулей
  • Конфигурации, настройки интеграций, доступы
  • Сопроводительная документация и инструкции по эксплуатации
  • Вводный инструктаж ответственного сотрудника Заказчика
  • Подписание акта сдачи-приёмки по всему объёму работ
